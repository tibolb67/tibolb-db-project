Use Case 1 – Erfassung einer Ausgabe

Kurzbeschreibung Die Nutzer*innen erfassen eine Ausgabe mit Betrag, Datum und Kategorie, die in der Datenbank gespeichert wird.
Beteiligte Akteure Nutzer*in
Auslöser Nutzer*in möchte eine Ausgabe eintragen
Vorbedingungen Nutzer*in muss eingeloggt sein
Normalablauf
  1. Nutzer*in klickt auf “Add” (Name kann verändert werden)
  2. Nutzer*in gibt Betrag, Datum und Kategorie ein
  3. Nutzer*in wählt einen Account
  4. Ausgabe wird gespeichert
Alternative Abläufe Kategorie existiert nicht (z.B. kein Essen oder Freizeit)
  1. Nutzer*in soll eine neue Kategorie herstellen
Daten fehlen (nicht eingetragen) z.B. kein Datum
  1. System fragt für eine Ergänzung
Ergebnis Ausgabe ist gespeichert und wird dem korrekten Account geordnet
Bemerkungen Wird später für Berechnungen oder Verlauf genutzt.
