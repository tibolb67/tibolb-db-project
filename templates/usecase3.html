Use Case 3 – Zahlungserinnerungen

Kurzbeschreibung Das System erinnert Nutzer*innen automatisch an offene Zahlungen.
Beteiligte Akteure Nutzer*in
Auslöser Eine Zahlung hat ihr Datum erreicht
Vorbedingungen Zahlung muss ein Fälligkeitsdatum haben, sodass die Erinnerung aktiviert wird
Normalablauf
  1. Nutzer*in fügt eine Zahlung mit Fälligkeitsdatum ein
  2. System prüft offene Zahlungen
  3. System sendet automatisch eine Zahlungserinnerung
Alternative Abläufe Fälligkeitsdatum fehlt oder ist falsch geschrieben (z.B. 13.26.2013)
  1. System fragt für ein neues Datum oder eine Ergänzung
Ergebnis Nutzer*in bekommt rechtzeitig eine Zahlungserinnerung
Bemerkungen Weniger vergessene oder verspätete Zahlungen
